## 💡 Novel Differentiators

Features that set cruise-ai apart from existing developer tooling:

| Feature | Why It Stands Out |
|---------|-------------------|
| **Teach Me vs Do It For Me** | Balances automation with learning instead of hiding implementation details |
| **"Why this, why not that?"** | Explains why a Skill is better than an MCP, or why a Hook is better than an Agent for a given workflow |
| **Automation Opportunity Detection** | Continuously mines workflows to identify what should become reusable automation |
| **Token Optimization Coach** | Goes beyond token counting by suggesting architectural improvements (memory, MCPs, model choice, prompt simplification) |
| **Artifact Recommendation Engine** | Recommends the right abstraction (Skill, Hook, MCP, Agent, Harness, Memory) instead of forcing users to decide |
| **Impact Measurement** | Tracks token savings, time savings, and adoption after recommendations are applied |
| **Automation Packs** | Solves a problem with a coordinated set of artifacts instead of generating a single file |
| **AI Engineering Learning Path** | Uses the user's actual workflow to progressively teach more advanced concepts |
